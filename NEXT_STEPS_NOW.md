### Step 4: Build Your Android App (30-60 minutes)

**You need to create a production build of your app.**

#### Option A: Using Expo/EAS Build (Recommended)

**First, install EAS CLI:**
```bash
npm install -g eas-cli
```

**Login to Expo:**
```bash
eas login
```

**Configure EAS (if not done):**
```bash
eas build:configure
```

**Build Android app:**
```bash
eas build --platform android --profile production
```

**This will:**
- Create an Android App Bundle (AAB file)
- Upload it to Expo's servers
- Give you a download link when done

**Time:** 30-60 minutes (mostly waiting for build to complete)

#### Option B: Using Expo Build (Legacy)

If you're using the older Expo build service:
```bash
expo build:android -t app-bundle
```

**Note:** This may require Expo account setup.

## üö® Common Issues & Solutions

### Issue: "Build failed"
- **Solution:** Check Expo/EAS Build logs for errors
- Make sure all dependencies are installed
- Check `package.json` for correct versions

### Issue: "Screenshots rejected"
- **Solution:** Make sure screenshots are clear and show your app
- Remove any personal information
- Use high-quality images

### Issue: "Content rating pending"
- **Solution:** Wait a few minutes, it usually completes quickly
- Check email for any questions from Google

### Issue: "Privacy Policy URL not accessible"
- **Solution:** Verify GitHub Pages is enabled
- Test URL in browser before submitting
